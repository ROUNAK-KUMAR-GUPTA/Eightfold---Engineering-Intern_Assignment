Multi-Source Candidate Data Transformer

Technical Design — Aman Kumar (aman.kumar.dev2025@gmail.com)

1. Problem & Objective

Candidate data arrives from heterogeneous sources—recruiter CSVs, resume PDFs, GitHub profiles, LinkedIn—each with different schemas, fields, and formats. The pipeline ingests these sources, normalizes fields to canonical forms, merges/deduplicates into one canonical profile per candidate, tracks provenance (source, method, confidence), and projects output at runtime via configurable layer—while gracefully degrading on missing/garbage input.

3. Data Model & Provenance

Canonical Profile (Pydantic v2)

  • candidate_id: MD5 of first match key
  • full_name; emails(lowercase,dedup); phones(E.164)
  • locations: List[{city, state, country(ISO α2)}]
  • skills: List[str] (canonical, deduped)
  • experience: List[{company, title, start, end, summary}]
  • education: List[{institution, degree, field, end_year}]
  • links: List[{type, url}]; websites: List[{type, url}]
  • years_experience: Opt[float]; summary: Opt[str]

Provenance per field

  • source: SourceType enum (recruiter_csv, resume_pdf, github_api)
  • method: direct|priority_pick|conflict_resolved|concatenated
  • raw_value: original before normalization
  • confidence: float [0,1]

Confidence Model

conf = source_priority × base; corroborate +0.1; conflict −0.2; clamped [0,1]

5. Merge Algorithm

  1. Match: Group by identity keys: email(exact)→phone(E.164)→norm-name; first match wins, transitive.
  2. Normalize: Apply all normalizers to each raw candidate before merge.
  3. Merge scalars: Highest-priority wins. Agreement→+0.1; disagreement→−0.2(conflict_resolved).
  4. Merge lists: Union+dedup; confidence=weighted avg of contributing sources.
  5. Provenance: Every field records (source, method, raw_value, confidence).
  6. ID: MD5 hex of first match key→determinism.

6. Projection Layer

Runtime JSON config reshapes output without code changes: fields (select, rename via output_name, nested paths like phones[0]); normalization (E164, canonical, lowercase, uppercase, trim); include_confidence ({value,confidence,sources} wrapper); on_missing (null|omit|error).

7. Robustness & Scalability

Graceful degradation: Missing sources→skip w/ warning; garbage rows→empty candidates; bad phones/dates→raw w/ low confidence. Determinism: Sorted processing; MD5 IDs from stable keys; byte-identical on repeated runs. Scalability: O(N) hash-grouped matching; streaming NDJSONL; parallelizable parsers. Extensibility: New sources→parser+priority; new normalizers→registry; projection→declarative JSON.
